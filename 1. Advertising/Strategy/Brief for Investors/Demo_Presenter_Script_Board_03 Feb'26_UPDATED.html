Step 1: ASIN Config (Central Hub)

1 min

Each row is a product. You can see: Target ACOS, Daily Budget, Auto Budget toggle, Auto Pacing, and the KW Research status.

The Target ACOS follows a hierarchy: ASIN level is the default, but you can override it at Campaign level or even Keyword level for granular control."

Step 2: Adding Product & Competitor Discovery

1.5 min
Click Add button → Product selection → Competitor Discovery page
Show the Add flow (or describe with screenshots if already done)
"To add a product, click Add, select from your catalog, then move to competitor discovery.

You have two options for finding competitors:
1. Manual entry — if you know your competitors

You need at least 5 competitors selected. You can mix both methods."

Step 3: AI Keyword Research (3 Tabs)

2.5 min
ASIN Config → Click "Generate Research" → then "View Research"
"Click 'Generate Research' to trigger AI-powered keyword analysis. Once complete, click 'View Research' to review.

The research has three tabs that need approval before campaigns are created:"

Tab 1: Branding Scope & KW-Product Relationship

Show Tab 1 with the classification table
"Tab 1 classifies keywords on two dimensions:

Branding Scope:
• NB = Non-Branded (generic keywords)
• OB = Own Branded (your brand name)
• CB = Competitor Branded

Relationship:
• R = Relevant (direct product keywords)
• S = Substitute (alternative products)
• C = Complementary (related products)
• N = Negative (keywords to exclude)

You can Export for bulk editing, make changes, Import back, then Submit for approval."

Tab 2: Listing Attributes Ranking

Click to Tab 2
"Tab 2 shows product attributes extracted from your listing — product type, variants, use cases.

You can adjust rankings to prioritize certain attributes in targeting. Review, modify if needed, Submit."

Tab 3: Keyword Groupings

Click to Tab 3
"Tab 3 is where keywords are grouped by customer search intent.

Each group becomes a campaign. For example:
• Group 1: 'Diffuser Oil + Italy variant' — all keywords matching this intent
• Group 2: 'Diffuser Oil + Lemon scent' — different intent cluster

Relevant keywords can have multiple groups. Negative, Substitute, Complementary get one group each.

Once you Submit this tab — campaigns are automatically generated based on these groupings."
Acknowledge Status: "The feature is fully built. Current challenge is output accuracy — results need manual review. We've completed 2/3 of quality improvements, deploying this week. If satisfactory, we reduce manual review. If not, we wait for the remaining 1/3."
"Now let's see the campaigns that were generated..."

Step 4: Campaign Config

1.5 min
Navigate to Campaign Config tab
Show the campaign list with auto-generated campaigns
"Here are the campaigns auto-generated from the keyword groupings.

The naming convention tells you everything:
• OB-R2 = Own Branded, Relevant keywords, Group 2
• SP-KW = Sponsored Product, Keyword targeting
• E at the end = Exact match

Each campaign shows:
• Target ACOS (optional — falls back to ASIN level if empty)
• Daily Budget (optional if Auto Budget is on)
• Status: PAUSED or ENABLED
• Targeting Spec: shows what customer intent this campaign targets"
Point to the Targeting Spec column
"This 'Targeting Spec' column is key — it tells you which attributes/intents this campaign covers. Helps you decide which campaigns to activate.

Important: The first time you Submit with Status=ENABLED, campaigns are actually created on Amazon. After that, toggling just changes status."
"Once campaigns are live, you get granular keyword control..."

Step 5: KWs Config (Keyword-Level Control)

1 min
Navigate to KWs Config tab
Show the keyword table with filters
"KWs Config gives you control over individual keywords across all campaigns.

Key features:
• Bid Controls: Set Fixed Bid (won't change), Ceiling (max), Floor (min) — with expiry dates
• Target ACOS override: Set keyword-specific targets
• Performance metrics: ACOS, TOS%, CTR, CVR across multiple time windows

Filter by ASIN or Campaign. Export for bulk changes, Import back, Submit."
This tab only populates AFTER campaigns are enabled for the first time

PART 3: AUTOMATION (Passive Features)

3-4 minutes

Automated Features Running in Background

3-4 min
"Behind the scenes, several automation features are working continuously:"

1. Bid Optimization

"System adjusts bids based on TOS% (Top of Search) and ACOS performance.
• Low TOS → increase bid to get better placement
• High TOS → decrease bid to save cost
• ACOS breach (spending badly) → emergency drop to $0.02

Uses CVR waterfall — looks at conversion data at multiple levels to make intelligent decisions even with sparse data."
Mention: CVR estimation accuracy is still being improved — affects bid optimization quality

2. Auto Budget Management

"Distributes ASIN-level budget across campaigns based on performance.
• Winners (ACOS ≤ target) get more budget
• Losers get reduced (max 20% reduction per cycle)
• Every campaign gets at least $5 minimum"

3. Harvesting & Negation

"Continuously optimizes targeting:
• Harvesting: High-performing search terms get promoted to dedicated campaigns
• Negation: Underperforming terms get added as negatives automatically"

4. Intelligent Pacing

"Decides which campaigns to activate and how to allocate budget.

Current release is a rule-based base model — establishes core workflow and validates direction.

Post-validation, we'll enhance with ML. The pacing workflow has multiple input/output chains — feedback from base model will guide which chain to enhance first."

5. High Performers Isolation

"Top 5% keywords by sales get their own 'solo' campaigns for maximum control. System auto-pauses duplicates to prevent self-competition."
