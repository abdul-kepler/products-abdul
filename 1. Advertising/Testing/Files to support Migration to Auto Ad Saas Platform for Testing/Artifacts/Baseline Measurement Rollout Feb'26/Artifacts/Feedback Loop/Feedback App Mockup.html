Automated Bid Optimization

Check Weekly

What This Feature Does

Calculates optimal bids based on target ACOS/ROAS and keyword performance, then pushes to Amazon.

‚úì
Non-performing keywords get bid decreases
Keywords with high ACOS (>target) should have lower bids than before
How to test:
1. Go to Portal ‚Üí Bid Strategy Logs
2. Filter by account, last 7 days
3. Sort by ACOS (highest first)
4. Check: Did high-ACOS keywords get bid decreases?
Pass if: Majority of high-ACOS KWs show decreased bids
Fail if: High-ACOS KWs kept same or higher bids
‚úì
Performing keywords get bid increases
Keywords with conversions and good ACOS should have higher bids
How to test:
1. Go to Portal ‚Üí Bid Strategy Logs
2. Filter by account, last 7 days
3. Sort by conversions (highest first)
4. Check: Did converting keywords get bid increases?
Pass if: Converting KWs with below-target ACOS show increased bids
Fail if: Converting KWs stayed flat despite room to grow
Bid values are within reasonable range
No extreme bids ($0.01 or $50+) unless justified
How to test:
1. Go to Portal ‚Üí Keyword Config
2. Sort by bid (lowest first, then highest)
3. Check: Any bids below $0.10 or above $20?
Pass if: All bids in reasonable CPC range for category
Fail if: Extreme values without justification
Bids are syncing to Amazon
Calculated bids should actually reach Amazon
How to test:
1. Go to Portal ‚Üí Bid Strategy Logs
2. Check "Sync Status" column
3. Spot-check 3 keywords in Amazon Seller Central
Pass if: Sync status = Success, Amazon bids match Portal
Fail if: Sync failures or mismatched bids
New keywords get appropriate starting bids
Newly added keywords should not start at extreme values
How to test:
1. Go to Portal ‚Üí Keyword Config
2. Filter by "Added date" = last 7 days
3. Check starting bid values
Pass if: Starting bids are reasonable (not min/max)
Fail if: All new KWs at floor or ceiling
Bid changes respect daily limits
Bids shouldn't jump dramatically in one day
How to test:
1. Go to Portal ‚Üí Bid Strategy Logs
2. Look at bid change history for any keyword
3. Check if any single-day change >50%
Pass if: Changes are gradual (‚â§30% per day typically)
Fail if: Wild swings without justification

How to Operate

Daily and weekly routines

Daily Routine (15-20 min)

1
Check Dashboard (2 min)
Open this app ‚Üí Dashboard. Check for any new critical issues overnight.
2

Go to UAT Checklists. Run the "Daily" checks for Budget Optimization and Pacing. Check the boxes as you go.

Weekly Routine (Friday, 30 min)

1
Complete Weekly Checks (15 min)
Run all "Weekly" UAT checks: Bid Optimization, Negation, Harvesting, KW Research.
2
Review All Feedback (10 min)
Go to "All Feedback". Review open items. Add any comments or updates.
3
Export Weekly Report (5 min)
Click "Export Weekly Report" on Dashboard. Send to Abdul in Slack.

Using AI-Assisted Feedback

Why Use AI Entry?

Faster than filling forms. Just describe what you saw in plain language - the AI will structure it properly.

1
Describe the Issue Naturally
Good: "Bid for 'dog casket' dropped from $1.50 to $0.15 but it had 3 conversions last week with 12% CVR. Account: PetCaskets."
Bad: "bid issue" or "something wrong with bids"
2
Include Key Details
Account name, specific keyword/campaign, numbers (before/after), what you expected.
3

AI will fill the form. Review and adjust if needed, then submit.

Known Limitations

Issues we're aware of - don't report these as new bugs

CVR Estimation Model

Current approach: Basic waterfall (Keyword CVR ‚Üí Campaign CVR ‚Üí Account CVR ‚Üí Default 5%)

Problem: Not ML-based. Doesn't account for seasonality, competition, listing changes.

Result: Under-bidding (CVR underestimated) or Over-bidding (CVR overestimated)

If you see this: Log it as "Known Limitation - CVR" so we track frequency. Not a bug to fix now.

Missing Keywords

Reasons:

  • 3rd party data source is weak (no immediate fix)
  • Logic issue for performing KWs (fix pending)

If you see this: Log as "Data Gap" with the specific keyword. Helps us track what's missing.

Portal Bugs (Known)

Bug Workaround
PROD-2666: Some targets missing in Bid Strategy Logs Use Keyword Config instead
PROD-2665: Export incomplete Use smaller date ranges
PROD-2573: Pagination stuck at 20 Use filters to narrow results

If you hit these: Don't report as new. Mark as "Known" if it blocks you significantly.

What this means: When performance drops, you need to investigate manually using the diagnosis flow in Guidelines.
