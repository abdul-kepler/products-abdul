Guidelines

Classification reference

Symptom vs Root Cause

SYMPTOM = What happened

Feature MisbehaviorFeature did something wrong
System ErrorSomething broken
UI/UX ProblemPortal usability
Data GapMissing/wrong data
Client ComplaintClient raised concern

ROOT CAUSE = Why it happened

Logic IssueAlgorithm wrong
Known LimitationCVR, data source
Manual Input ErrorWrong pacing/budget
System BugCode error
External FactorCompetition, client
UnknownNeeds investigation

UAT vs Feedback Form

UAT Checklists Feedback Form
When Scheduled testing (daily/weekly) Anytime - ad hoc
What Predefined test cases Anything else
How Pass/Fail buttons ‚Üí Fail auto-creates feedback AI-assisted or manual form
Result Can be any symptom/root cause depending on test You select symptom + root cause
