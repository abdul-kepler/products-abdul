1. Automated Bid Optimization
Tests for bid calculation accuracy, CVR estimation, and bid adjustments

Core Formula

Bid = Target_ACOS x CVR x AOV (Price)

CVR Waterfall (Manual Exact)

Level 1: ASIN + Search Term (Confidence: 1.0)
Level 2: ASIN + Tag (Confidence: 0.95)
Level 3: ASIN Level (Confidence: 0.90)
Level 4: Default (Confidence: 0.80)
P Performance Tests (ACOS & Sales Impact)
BID-P01
ACOS Critical
High ACOS Detection - Bid Too High
Verify system reduces bid when ACOS exceeds target, leading to improved efficiency
Steps
  1. Find keyword with ACOS > Target_ACOS + 10%
  2. Check current bid vs calculated bid using formula
  3. Wait for bid optimization cycle (or trigger manually)
  4. Verify bid decreased appropriately (10-20% reduction expected)
Pass Criteria: Bid reduced by 10-20%. New projected ACOS closer to target.
ACOS Impact: Decrease
Sales Impact: May decrease short-term
BID-P02
Sales Critical
Low Sales Detection - Bid Too Low
Verify system increases bid when ACOS is well below target (leaving money on table)
Steps
  1. Find keyword with ACOS < Target_ACOS - 15% AND has impression share headroom
  2. Calculate optimal bid: Target_ACOS x CVR x Price
  3. Verify current bid is significantly below optimal
  4. Wait for optimization cycle
  5. Verify bid increased toward optimal
Pass Criteria: Bid increased. Impression share should improve. Sales should increase while ACOS stays within target.
Sales Impact: Increase
ACOS Impact: May increase but within target
BID-P03
ACOS Sales
Optimal Bid Maintenance
Verify system maintains bid when ACOS is within acceptable range of target
Steps
  1. Find keyword with ACOS within +/-5% of Target_ACOS
  2. Record current bid
  3. Wait for multiple optimization cycles
  4. Verify bid remains stable (no unnecessary changes)
Pass Criteria: Bid remains unchanged or changes minimally (<5%). System doesn't over-optimize stable performers.
F Functional Tests (Logic Verification)
BID-F01
Critical
CVR Waterfall Level 1 - ASIN + Search Term
Verify CVR is correctly estimated using most granular data when available
Steps
  1. Find keyword with sufficient data at ASIN + Search Term level (>= X clicks threshold)
  2. Verify system uses Level 1 CVR (ASIN + ST specific)
  3. Verify confidence score = 1.0
  4. Calculate expected bid and compare to actual
Pass Criteria: Bid calculated using ASIN+ST level CVR. Bid Strategy Logs show Level 1 used. Confidence = 1.0.
BID-F02
Critical
CVR Waterfall Fallback - Level 2, 3, 4
Verify system correctly falls through waterfall when higher levels lack data
Steps
  1. Find new keyword with no search term level data
  2. Verify system falls to Level 2 (ASIN + Tag)
  3. If no tag data, verify falls to Level 3 (ASIN)
  4. If no ASIN data, verify falls to Level 4 (Default)
  5. Check confidence scores match waterfall level
Pass Criteria: Correct waterfall level selected. Confidence decreases appropriately (0.95 -> 0.90 -> 0.80).
BID-F03
Critical
Target ACOS Hierarchy
Verify correct Target ACOS is used (ASIN-level > Global)
Steps
  1. Set ASIN-level Target ACOS = 25%
  2. Set Global Target ACOS = 30%
  3. Verify bid calculation uses 25% (ASIN-level)
  4. Remove ASIN-level target
  5. Verify bid calculation now uses 30% (Global)
Pass Criteria: System respects ASIN-level Target ACOS when set. Falls back to Global when ASIN-level not set.
BID-F04
UI
Bid Strategy Logs Accuracy
Verify Bid Strategy Logs UI shows correct calculation details
Steps
  1. Navigate to Bid Strategy Logs
  2. Filter by specific keyword
  3. Verify displayed: CVR used, CVR waterfall level, Target ACOS, Price, Calculated Bid
  4. Manually verify calculation: Bid = Target_ACOS x CVR x Price
Pass Criteria: All components displayed correctly. Manual calculation matches system calculation.
E Edge Cases & Boundary Conditions
BID-E01
Critical
Zero CVR Handling
Verify system handles keywords with zero conversions correctly
Steps
  1. Find keyword with clicks but zero orders (CVR = 0)
  2. Verify system doesn't calculate Bid = 0
  3. Verify system uses waterfall fallback or default CVR
Pass Criteria: System uses fallback CVR from higher waterfall level. Bid is reasonable, not zero.
BID-E02
Critical
Extremely High CVR Outlier
Verify system handles statistical outliers (e.g., 1 click, 1 order = 100% CVR)
Steps
  1. Find keyword with very few clicks but high CVR (e.g., 2 clicks, 2 orders)
  2. Verify system doesn't over-bid based on statistically insignificant data
  3. Check if confidence score or minimum threshold prevents outlier impact
Pass Criteria: System applies confidence adjustment or uses higher waterfall level for insufficient data.
BID-E03
ACOS
Bid Floor/Ceiling Enforcement
Verify system respects minimum and maximum bid limits
Steps
  1. Calculate scenario where formula produces bid below Amazon minimum ($0.02)
  2. Verify system sets bid to minimum, not calculated value
  3. Calculate scenario where formula produces extremely high bid
  4. Verify system has reasonable ceiling or warning
Pass Criteria: Bids respect platform minimums. Excessive bids are capped or flagged.
N Negative Tests (What Should NOT Happen)
BID-N01
Critical
Bid Should Not Change for Locked/Manual Override Keywords
Verify manually set bids are not overwritten by automation
Steps
  1. Set a manual bid override on a keyword
  2. Wait for optimization cycles
  3. Verify bid remains at manually set value
Fail if: Manually set bid is changed by automation without user action.
BID-N02
Critical
Bid Should Not Use Stale Data
Verify bid calculations use fresh, not cached data
Steps
  1. Note current CVR and bid for a keyword
  2. Wait for significant performance change (new orders)
  3. Verify next optimization cycle uses updated CVR
Fail if: Bid calculation uses outdated CVR data after data refresh.
2. Automated Budget Optimization
Tests for budget allocation, knapsack optimization, and ACOS-based prioritization

Priority Logic

If ACOS_proj <= Target_ACOS: Priority = "Favorable"
If ACOS_proj > Target_ACOS: Priority = "Unfavorable"

Knapsack Optimization

Maximize: Total Projected Sales
Constraint: Total Spend <= Daily Budget
Filter: ACOS_proj <= Target_ACOS (Favorable tier first)
P Performance Tests
BUD-P01
Sales Critical
Budget Utilization Efficiency
Verify budget is fully utilized on high-potential campaigns
Steps
  1. Set daily budget = $100
  2. Ensure multiple campaigns with varying projected ACOS
  3. Run budget optimization
  4. Verify total allocated budget approaches $100
  5. Verify favorable ACOS campaigns are prioritized
Pass Criteria: Budget utilization > 90%. Favorable campaigns receive budget before unfavorable ones.
Sales Impact: Maximized within budget
ACOS Impact: Maintained within target
BUD-P02
ACOS Critical
Unfavorable Campaign Deprioritization
Verify campaigns with projected ACOS above target get lower priority
Steps
  1. Create scenario with limited budget
  2. Have Campaign A: proj ACOS = 20% (Target: 25%) - Favorable
  3. Have Campaign B: proj ACOS = 35% (Target: 25%) - Unfavorable
  4. Verify Campaign A gets full budget allocation first
  5. Campaign B only gets remaining budget (if any)
Pass Criteria: Favorable campaigns fully funded before unfavorable. Overall portfolio ACOS stays closer to target.
F Functional Tests
BUD-F01
Critical
Knapsack Solution Optimality
Verify knapsack algorithm selects optimal campaign mix
Steps
  1. Create test scenario with 5 campaigns, known projected sales/spend
  2. Manually calculate optimal selection for given budget
  3. Run budget optimizer
  4. Compare system selection to manual calculation
Pass Criteria: System selection matches or exceeds manual optimal solution in total projected sales.
BUD-F02
UI
Budget Allocation Visibility
Verify UI shows budget allocation reasoning
Steps
  1. Navigate to budget optimization UI
  2. Verify each campaign shows: allocated budget, projected ACOS, priority tier
  3. Verify total allocated vs available budget is displayed
Pass Criteria: User can understand why each campaign received its budget allocation.
E Edge Cases
BUD-E01
Critical
All Campaigns Unfavorable
Verify system behavior when no campaigns meet ACOS target
Steps
  1. Create scenario where all campaigns have proj ACOS > Target
  2. Run budget optimizer
  3. Verify system still allocates budget (doesn't leave 100% unspent)
  4. Verify it selects least-bad options (closest to target)
Pass Criteria: System allocates to campaigns closest to target. Doesn't waste entire budget.

Critical
Budget Exceeds Capacity

3. Intelligent Pacing
Tests for campaign activation, pause/resume logic, and projection accuracy

Pacing States

AUTO_ACTIVE: System activated, can be paused by system
AUTO_PAUSED: System paused, can be resumed by system
LOCKED_ACTIVE: Manual override, system won't pause
LOCKED_PAUSED: Manual override, system won't resume

Projection Chain

CVR estimate -> CTR estimate -> Impression Share -> Projected Clicks -> Projected Spend/Sales
P Performance Tests
PAC-P01
Sales Critical
High-Potential Campaign Activation
Verify system activates campaigns with favorable projections
Steps
  1. Create campaign with projected ACOS well below target
  2. Set campaign to AUTO_PAUSED state
  3. Ensure budget availability
  4. Run pacing engine
  5. Verify campaign transitions to AUTO_ACTIVE
Pass Criteria: Campaign activated. Actual performance should align with projections over time.
Sales Impact: Capture missed revenue
ACOS Impact: Within target
PAC-P02
ACOS Critical
Poor-Performing Campaign Pause
Verify system pauses campaigns exceeding ACOS thresholds
Steps
  1. Find campaign with actual ACOS significantly above target
  2. Verify campaign is AUTO_ACTIVE
  3. Run pacing engine
  4. Verify campaign transitions to AUTO_PAUSED
  5. Verify budget reallocated to better performers
Pass Criteria: Campaign paused. Overall portfolio ACOS improves. Budget not wasted.
PAC-P03
Sales
Campaign Resume After Performance Improves
Verify system resumes paused campaigns when conditions improve
Steps
  1. Find AUTO_PAUSED campaign
  2. Simulate improved CVR data (e.g., through organic improvements)
  3. Wait for pacing engine re-evaluation
  4. Verify campaign is reconsidered for activation
Pass Criteria: Campaign reactivated when projections become favorable. Not permanently blacklisted.

Daily Pacing Trigger

  4. Verify state changes logged with correct timestamps

PAC-N01
Critical
Rapid State Flipping Prevention
Verify system doesn't rapidly pause/resume the same campaign

4. Automated Negation
Tests for negative keyword identification and application

Trigger Frequency

Runs every 3 hours (changed from 24 hours)

Negation Criteria

Search term qualifies for negation if:
Spend >= Threshold AND Conversions = 0 AND Clicks >= Min_Clicks
OR
ACOS > Target_ACOS * 2 AND Spend >= Threshold

  3. Verify term is added as negative keyword
  4. Verify spend on this term stops
Pass Criteria: Term negated. Future spend on term = $0. ACOS should improve as waste eliminated.
ACOS Impact: Decrease (waste removed)
Sales Impact: Budget redirected to converters

5. Automated Harvesting
Tests for search term graduation to exact match campaigns

Harvesting Qualification

MAX(ACoS_7d, ACoS_15d, ACoS_30d, ACoS_90d) <= Target_ACoS
AND Lifetime_Ad_Spend(ASIN + Search_Term) >= ASIN's Price

Zero-Sales Handling

If Ad_Sales = 0 AND Ad_Spend > 0: Set ACoS = 9999 (auto-fail qualification)
P Performance Tests
HAR-P01
Sales Critical
Proven Winner Harvesting
Verify system harvests search terms with consistent profitability
Steps
  1. Find search term with: ACOS_7d, 15d, 30d, 90d all < Target
  2. Verify Lifetime Spend >= ASIN Price
  3. Wait for harvesting cycle (24 hours)
  4. Verify exact match keyword created in appropriate campaign
  5. Verify keyword configured correctly (bid, targeting)
Pass Criteria: Exact match keyword created. Better bid control on proven performer. Should improve ACOS and capture more sales.
Sales Impact: Increase (better targeting)
ACOS Impact: Maintain or improve
HAR-P02
ACOS
MAX ACOS Check (Worst Period)
Verify system uses MAX across all time windows, not just recent
Steps
  1. Find term with: ACOS_7d=15%, ACOS_90d=35% (Target=25%)
  2. Verify term is NOT harvested (90d fails)
  3. Wait for 90d ACOS to improve below target
  4. Then verify harvesting occurs
Pass Criteria: System checks all time windows. Doesn't harvest terms with recent spike but poor history.
F Functional Tests
HAR-F01
Critical
Minimum Spend Threshold
Verify system requires sufficient spend before harvesting
Steps
  1. Find term with great ACOS but Spend < ASIN Price
  2. Verify term is NOT harvested yet
  3. Wait until spend exceeds ASIN price
  4. Verify harvesting triggers
Pass Criteria: System waits for statistical significance. No premature harvesting.
HAR-F02
Critical
ASIN vs Keyword Harvesting Routing
Verify ASIN search terms route to ASIN campaigns, keywords to keyword campaigns
Steps
  1. Find qualified search term that is a 10-digit ASIN
  2. Verify it creates targeting in ASIN campaign (not keyword campaign)
  3. Find qualified search term that is a keyword phrase
  4. Verify it creates exact match in keyword campaign
Pass Criteria: Correct routing: ASINs -> ASIN campaigns, Keywords -> Keyword campaigns.
N Negative Tests
HAR-N01
Critical
Zero-Sales Term Exclusion
Verify zero-sales terms are auto-excluded (ACOS = 9999)
Steps
  1. Find term with $50 spend, 0 sales
  2. Verify system calculates ACOS = 9999 (not infinity/error)
  3. Verify term is NOT harvested
Fail if: Zero-sales term is harvested. Would waste more money.
HAR-N02
Critical
No Duplicate Harvesting
Verify already-harvested terms are not harvested again
Steps
  1. Confirm a term has already been harvested
  2. Wait for multiple harvesting cycles
  3. Verify no duplicate keyword created
Fail if: Duplicate keyword created in same or different campaign.
