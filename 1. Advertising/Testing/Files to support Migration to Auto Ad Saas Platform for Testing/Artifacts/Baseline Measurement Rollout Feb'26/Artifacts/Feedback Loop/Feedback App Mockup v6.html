The Two Core Problems

Every issue during rollout maps to one of these two problems. UAT tests are designed to catch causes before they become problems.

Problem 1: Poor Efficiency (High ACOS)

Spending money but not getting good returns

Symptoms:
  • ACOS above target
  • High spend, low conversions
  • Client complaints about wasted spend

Problem 2: Low Sales

Not selling enough / missing revenue opportunity

Sub-types:
  • Low Spend - Can't spend the budget
  • Same Spend, Worse Results - Spending but on wrong things

Problem → Root Cause Map

Each problem has multiple potential root causes. UAT tests verify each root cause is working correctly.

The Bid Formula - Source of Truth

Most efficiency and sales issues trace back to this formula. Understanding it is key to testing.

Bid = Target_ACOS × CVR × AOV Where: Target_ACOS = Client's goal ACOS (e.g., 30% = 0.30) CVR = Conversion Rate estimate (from waterfall) AOV = Average Order Value Example: Target_ACOS = 0.30 (30%) CVR = 0.05 (5% conversion rate) AOV = $50 Bid = 0.30 × 0.05 × $50 = $0.75

CVR Waterfall (When keyword-level CVR unavailable)

Level Data Source Confidence When Used
1 (Best) ASIN + Keyword + Match Type 1.0 Keyword has conversion history
2 ASIN + Tag + Match Type 0.8 No keyword data, use tag average
3 ASIN + Match Type 0.6 No tag data, use ASIN average
4 (Worst) Default CVR 0.4 New ASIN, no historical data
Key Testing Insight: If CVR is wrong, bid is wrong, and either efficiency OR sales suffers.
  • CVR too high → Bid too high → High ACOS (efficiency problem)
  • CVR too low → Bid too low → Low impressions → Low sales

E1
High ACOS Keyword - CVR Mismatch Check
Critical

Scenario: A keyword has ACOS significantly above target. Is the system aware and responding?

High ACOS → Bid Too High? → CVR Overestimated?
How to Test:
  1. Go to Keyword Config → Sort by ACOS descending
  2. Pick the TOP 3 highest ACOS keywords (above target)
  3. For each keyword, open Bid Strategy Logs
  4. Record these values:
    • Current Bid: $____
    • CVR Used: ____% (from which level?)
    • Target ACOS: ____%
    • Actual ACOS: ____%
  5. Calculate expected bid: Target_ACOS × CVR × AOV
  6. Compare: Is actual bid close to calculated bid?
  7. Check CVR source: Is the CVR estimate reasonable given actual performance?
Possible Root Causes if FAILING:
CVR from waterfall Level 2-4 is much higher than actual keyword CVR
CVR data is stale (not refreshed recently)
Manual bid override in place (check if bid is locked)
Bid change limits preventing necessary decrease
PASS if:
  • Bid matches formula (within 10%)
  • CVR source is appropriate
  • If high ACOS persists, bid is decreasing over time
FAIL if:
  • Bid doesn't match formula significantly
  • CVR clearly wrong (10% CVR used but 0 conversions in 30 days)
  • Bid not decreasing despite poor performance
E2
Negation - Poor Performer Removal Speed
High

Scenario: Search terms with high spend and no conversions should be negated. Is it happening?

High ACOS → Bad search terms still active → Negation too slow?
How to Test:
  1. Go to Search Term Report → Filter: Last 14 days
  2. Sort by Spend descending
  3. Find search terms with: Spend > $20 AND Conversions = 0
  4. For each, check:
    • Has it been negated? (Check Negative Keywords list)
    • When was last negation cycle? (Should be every 3 hours)
    • Does it meet negation criteria?
  5. If NOT negated, investigate why (criteria not met? bug?)
Possible Root Causes if FAILING:
Negation trigger not running (check last_run timestamp)
Criteria thresholds too lenient
Search term doesn't meet minimum impressions/clicks threshold
Negation disabled for this ASIN/campaign
PASS if:
  • Poor performers are negated within 24-48h
  • Negation cycle running every 3 hours
  • Clear reason why any non-negated term isn't meeting criteria
FAIL if:
  • High-spend zero-conversion terms active for 3+ days
  • Negation cycle not running
  • No explanation for why term isn't negated
E3
Budget Allocation - Worst Campaigns Check
High

Scenario: Budget should flow to best ACOS campaigns first. Are worst campaigns getting too much?

High ACOS → Budget going to bad campaigns → Knapsack allocation wrong?
How to Test:
  1. List all ACTIVE campaigns for an ASIN with their:
    • Campaign Name
    • ACOS (last 7 days)
    • Daily Budget Allocated
    • Actual Spend
  2. Sort by ACOS (worst first)
  3. Check: Is the WORST ACOS campaign getting significant budget?
  4. Compare to BEST ACOS campaign - is it getting MORE budget?
  5. Verify economic prioritization is working:
    • Campaigns with ACOS < Target should get priority
    • Campaigns with ACOS > Target should be deprioritized
Possible Root Causes if FAILING:
Knapsack not running (budget allocation stale)
Projection data wrong (predicting good performance for bad campaign)
Campaign manually locked ACTIVE
Only one campaign, so no choice
PASS if:
  • Best ACOS campaigns get more budget
  • Worst ACOS campaigns are deprioritized (less budget or paused)
  • Clear economic rationale for allocation
FAIL if:
  • Worst campaign getting most budget
  • No correlation between ACOS and budget allocation
  • Good campaigns starved while bad ones funded
E4
Pacing - Wrong Campaigns Active Check
Medium

Scenario: Pacing decides which campaigns to activate/pause. Is it choosing correctly?

How to Test:
  1. Go to Pacing Management page for an ASIN
  2. List all campaigns with their pacing state:
    • AUTO_ACTIVE / AUTO_PAUSED / LOCKED_ACTIVE / LOCKED_PAUSED
  3. For each AUTO_ACTIVE campaign, verify it SHOULD be active:
    • Has budget allocated?
    • Has reasonable projected ROI?
  4. For each AUTO_PAUSED campaign, verify it SHOULD be paused:
    • Was it deprioritized by knapsack?
    • Is there a clear reason for pause?
PASS if:
  • Active campaigns are economically viable
  • Paused campaigns have clear reason
  • Pacing states match knapsack decisions
FAIL if:
  • High-ACOS campaign active while low-ACOS paused
  • Pacing state doesn't match budget allocation
  • No clear logic for active/paused decisions
E5
Bid Decrease - Response to Poor Performance
Critical

Scenario: When a keyword performs poorly, does the bid decrease appropriately?

How to Test:
  1. Find a keyword that has had HIGH ACOS for 5+ consecutive days
  2. Go to Bid Strategy Logs for this keyword
  3. Look at bid history over the past 7 days:
    • Day 1 Bid: $____
    • Day 7 Bid: $____
    • % Change: ____
  4. Verify: Has the bid DECREASED in response to poor ACOS?
  5. If bid is flat/increasing despite poor ACOS, investigate:
    • Is bid at minimum already?
    • Is there a daily change limit blocking decrease?
    • Is CVR estimate not updating?
PASS if:
  • Bid decreased over the period
  • Or bid at minimum with clear documentation
  • CVR estimate updated to reflect poor performance
FAIL if:
  • Bid flat or increased despite poor ACOS
  • CVR estimate unchanged despite zero conversions
  • No explanation for lack of response

Path A: Low Spend (Can't spend the budget)

S1
Low Impressions - CVR Underestimate Check
Critical

Scenario: Keywords getting few impressions might have bids too low because CVR is underestimated.

Low Sales → Low Impressions → Bid Too Low? → CVR Underestimated?
How to Test:
  1. Go to Keyword Config → Sort by Impressions ascending (lowest first)
  2. Pick keywords with: Impressions < 100 in last 7 days but historically good
  3. For each, check in Bid Strategy Logs:
    • Current Bid: $____
    • CVR Used: ____% (from which level?)
    • Historical CVR (before migration): ____%
  4. Compare: Is current CVR estimate much LOWER than historical?
  5. Check competitor bids: Is our bid competitive for this keyword?
Possible Root Causes if FAILING:
CVR defaulting to low Level 3-4 value instead of historical data
New keyword with no history - using conservative default
Historical data not migrated properly
Bid at minimum due to budget constraints
PASS if:
  • CVR estimate reasonable for keyword history
  • Low impressions have clear reason (new kw, low search volume)
  • Bids competitive for the market
FAIL if:
  • Historically converting keyword now has 0.5% default CVR
  • Bid significantly lower than pre-migration
  • CVR not reflecting actual performance data
S2
Missing Keywords - Pre-Migration Comparison
Critical

Scenario: Keywords that drove sales before migration should still be active after migration.

Low Sales → Missing traffic sources → Key keywords not migrated?
How to Test:
  1. Get the TOP 20 keywords by Sales from BEFORE migration (Amazon Ads report, last 30 days pre-migration)
  2. For each keyword, search in Keyword Config in our system
  3. Create a checklist:
    KeywordPre-Migration SalesIn System?Active?Current Bid
    [keyword 1]$____Y/NY/N$____
    [keyword 2]$____Y/NY/N$____
    ...
  4. Calculate: What % of top 20 keywords are active in our system?
PASS if:
  • 90%+ of top pre-migration keywords are in system
  • All are active with competitive bids
  • Any missing have documented reason
FAIL if:
  • Key revenue keywords missing from system
  • Keywords in system but paused/inactive
  • No explanation for missing keywords
S3
Pacing - Good Campaigns Incorrectly Paused
High

Scenario: Pacing might have paused campaigns that were actually performing well.

How to Test:
  1. Go to Pacing Management → Filter: AUTO_PAUSED campaigns
  2. For each paused campaign, check:
    • Historical ACOS (before paused): ____%
    • Historical Sales (before paused): $____
    • Why was it paused? (Check pacing logs)
  3. Identify any campaigns that:
    • Had ACOS BELOW target when paused
    • Were driving significant sales
    • Should NOT have been paused based on economics
PASS if:
  • All paused campaigns had poor ACOS or low ROI
  • Clear economic reason for each pause
  • No high-performing campaigns paused
FAIL if:
  • Campaign with ACOS below target is paused
  • Top sales campaign paused while worse ones active
  • No clear reason for pause decision
S4
Bid Sync - Data Reaching Amazon
Critical

Scenario: Our calculated bids must actually reach Amazon. Sync failures = system not working.

Low Sales → System bids not on Amazon → Sync Failure
How to Test:
  1. Go to Bid Strategy Logs → Check Sync Status column
  2. Filter for: Sync Status = FAILED or PENDING for >1 hour
  3. Count: How many keywords have sync issues?
  4. For any failed syncs:
    • What's the error message?
    • When did it last try?
    • Is retry happening?
  5. Spot check 5 keywords: Compare Portal bid to Amazon Seller Central bid
PASS if:
  • 95%+ of bids synced successfully
  • Portal bids match Amazon bids (spot check)
  • Any failed syncs have clear error and retry
FAIL if:
  • Significant sync failures (>5%)
  • Portal and Amazon bids don't match
  • Failed syncs not retrying
S5
Budget Allocation - Best Campaigns Starved
High

Scenario: Best performing campaigns should get budget priority. Are they being starved?

How to Test:
  1. List all campaigns for an ASIN sorted by ACOS (best first)
  2. For the TOP 3 lowest ACOS campaigns:
    • Daily budget allocated: $____
    • Actual spend yesterday: $____
    • % of budget spent: ____
    • Limited by budget? Y/N
  3. Check: Are best campaigns fully utilizing their budget?
  4. If limited, check: Why not give them more budget?
PASS if:
  • Best campaigns spending their full allocation
  • If not spending, it's due to market/competition (not our limits)
  • Budget re-allocated from poor to good campaigns
FAIL if:
  • Best campaign has $10 budget while worse has $100
  • Good campaigns limited by budget while bad ones overfunded
  • Total budget available but not flowing to best

Harvesting - Converting Search Terms Captured
Medium

Scenario: Search terms that convert should be harvested into Manual campaigns for better control.

How to Test:
  1. Go to Search Term Report → Filter: Last 30 days, Conversions > 0
  2. Sort by Conversions descending
  3. For TOP 10 converting search terms:
    • Is it a keyword in our Manual campaigns?
    • If not, should it have been harvested?
    • Check: Does it meet harvesting criteria?
  4. For ASIN-based search terms (10-digit B0...):
    • Is it added to ASIN Targeting campaigns?
    • Does it show "Harvested" in System Remarks?
PASS if:
  • 80%+ of qualifying search terms are harvested
  • ASIN search terms in ASIN Targeting
  • Harvested items marked in UI
FAIL if:
  • High-converting search terms not in Manual campaigns
  • Harvesting criteria met but not harvested
  • ASIN search terms not becoming targets

Bid Increase - Response to Good Performance
High

Scenario: Keywords with ACOS below target should get bid increases to capture more volume.

How to Test:
  1. Find keywords with ACOS significantly BELOW target for 5+ days
  2. Check bid history in Bid Strategy Logs:
    • Day 1 Bid: $____
    • Day 7 Bid: $____
    • Direction: Increased / Flat / Decreased
  3. Verify: Is the system increasing bids to capture more volume?
  4. Check CVR: Has CVR estimate increased to reflect good performance?
PASS if:
  • Bid increased for high-performing keywords
  • CVR estimate updated to reflect conversions
  • System is aggressive on winners
FAIL if:
  • Great keyword stuck at low bid
  • CVR not updating despite conversions
  • Missing opportunity to scale winners

Test Methodology

How to approach each test case systematically.

1. Start With Outcomes, Not Features

Wrong approach: "Let me check if Bid Optimization is working"
Right approach: "Let me check if we have any High ACOS keywords and trace why"

2. Use Real Data, Not Sample Data

For each test:

  • Use ACTUAL keywords from the account being tested
  • Look at REAL performance data (last 7-14 days)
  • Compare to ACTUAL pre-migration benchmarks

3. The Investigation Flow

  1. Identify the symptom: High ACOS keyword, low impressions, missing keyword, etc.
  2. Trace the cause: What input/logic/data led to this outcome?
  3. Verify the logic: Given the inputs, is the output correct?
  4. Document the finding: Pass (working as intended) or Fail (bug/issue)

4. Example Investigation

Example: "Keyword 'dog casket' has 85% ACOS (target is 30%)"
Investigation steps:
  1. Check current bid: $2.50
  2. Check CVR used: 8% (from Tag level)
  3. Check AOV: $75
  4. Calculate expected bid: 0.30 × 0.08 × $75 = $1.80
  5. Compare: Actual bid ($2.50) > Expected ($1.80)
  6. Why? Check if bid is manually locked or change limit preventing decrease
  7. Check actual CVR: 2% (not 8%!)
  8. Root cause: CVR estimate (8%) much higher than actual (2%)
  9. Verdict: FAIL - CVR waterfall using stale tag data

5. When to Report a FAIL

Report as FAIL when... Do NOT report as FAIL when...
System behavior doesn't match documented logic Performance is suboptimal due to market conditions
Data inputs are clearly wrong (CVR way off) New keyword has conservative defaults (expected)
Feature not running when it should be Feature working but results take time
Sync failures blocking operations One-off sync retry that succeeded

6. Testing Schedule

Test Type Frequency Why
Sync checks (S4) Daily Sync failures are critical - catch immediately
CVR/Bid checks (E1, S1) Every 3 days Need time for performance data to accumulate
Pacing/Budget checks (E3, E4, S3, S5) Weekly Pacing decisions need time to show impact
Migration check (S2) Once at start, once at end Should be stable after initial setup
Negation/Harvesting (E2, S6) Weekly Need enough data for meaningful assessment
