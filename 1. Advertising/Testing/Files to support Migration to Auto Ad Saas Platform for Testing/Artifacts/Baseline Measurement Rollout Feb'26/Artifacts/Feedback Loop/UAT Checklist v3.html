1. Automated Bid Optimization
Core bid calculation, CVR/CTR waterfall models, and targeting-specific logic

Core Bid Formula

Bid = Target_ACOS x CVR x Price (AOV)

Fallback Hierarchy:
1. Targeting-type-level CVR from bid optimization
2. Account-level average CVR (if targeting type has no data)
3. System default CVR (5%) as last resort

Weak Fallback: >= 1 click (180 days only)
CTR Ceiling: 10% (0.10) cap

2. Automated Budget Optimization
Knapsack optimization, ACOS threshold filtering, budget allocation

Budget Allocation Algorithm

Available_Budget = Total_Budget - Reserved_Budget
Reserved_Budget = Sum(daily_budget) for all LOCKED_ACTIVE campaigns

Knapsack Optimization:
- Input: Campaigns filtered by ACOS threshold
- Objective: Maximize total projected sales within budget
- Constraint: Total spend <= Available_Budget

ACOS Threshold Filter

If ACOS_proj <= Target_ACOS:
    Priority_Tier = "Favorable"
Else:
    Priority_Tier = "Unfavorable"
    Sort by ACOS ascending (least-bad first)

Projection Formulas

Impressions_proj = available_daily_impressions x IS_estimate
Clicks_proj = Impressions_proj x CTR_estimate
Spend_proj = Clicks_proj x Bid
Sales_proj = Clicks_proj x CVR_estimate x ASIN_Price
ACOS_proj = Spend_proj / Sales_proj (if Sales_proj > 0)

Risk-Adjusted Sales Calculation

Keyword_Confidence = 0.5 x CVR_Confidence + 0.5 x CTR_Confidence
Campaign_Confidence = Sales-weighted avg of keyword confidences
Risk_Adjusted_Sales = Sales_proj x Campaign_Confidence

7. Automated Target Product Research
ASIN discovery, qualification, and brand grouping for ASIN Targeting

ASIN Qualification Logic - Level 1 (Qualified)

IF (Our.PricePerUOM <= Competitor.PricePerUOM)
AND (Our.Rating > 4
     OR (Our.Rating > Competitor.Rating - 0.5
         AND FirstDigit(Our.Rating) == FirstDigit(Competitor.Rating)))
AND ((Our.Reviews < 1000 AND Competitor.Reviews > 1000 AND Our.Reviews >= 0.1 * Competitor.Reviews)
     OR (Our.Reviews < 1000 AND Competitor.Reviews < 1000)
     OR (Our.Reviews > 1000))
THEN Qualification_Level = "Qualified_Level1"

ASIN Qualification Logic - Level 2 (Near-Qualified)

ELSE IF (Our.PricePerUOM <= 1.1 * Competitor.PricePerUOM)
AND (Our.Rating >= 4 OR Our.Rating >= Competitor.Rating - 0.3)
AND ((Our.Reviews > 100 AND Our.Reviews >= 0.1 * Competitor.Reviews)
     OR Our.Reviews < 100)
THEN Qualification_Level = "Qualified_Level2"

Data Points Collected

  • Price: Buy Box Price -> FBA Price -> FBM Price
  • Rating: Average customer rating
  • Review Count: Total reviews
  • Fulfillment Type: FBA or FBM
  • Brand Name

Brand Grouping

Group qualified ASINs by brand_name
Each brand group = one ASIN Targeting campaign
Store: brand_name, asin_list, asin_count, group_creation_date

8. Automated Keyword Research
Keyword discovery, relevancy scoring, and filtering

Relevancy Score Formula

Combined_Score = [(0.40 x Semantic) + (0.35 x Tag) + (0.25 x JS)] x Match_Type_Modifier

Match Type Modifiers:
- Exact: 1.00
- Phrase: 0.85
- Broad: 0.70

Tag-Based Decay: Base - (Tier - 1) x 0.02, floor 0.10
JS Relevancy: Normalized to 0-1 scale (divide by 100)

Data Quality Threshold

Keywords with 3+ null fields out of 13 -> Flag for review
