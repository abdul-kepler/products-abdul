What Makes an Issue "Critical"?

System auto-classifies based on these criteria

Criteria Why It's Critical Example
Sync Failure Data not reaching Amazon = feature doesn't work Bids calculated but not synced
Data Loss/Corruption Historical data gone or incorrect Keyword config wiped
Wrong Direction at Scale System doing opposite of intended across many items All bids increasing when should decrease
Client Escalation Client explicitly escalated to management Client threatening to leave
Everything else is Non-Critical: Logic issues, performance suboptimal, missing data, UI bugs, suggestions - these are tracked but don't block Go decision.

Guidelines

How this system works

Severity Classification (Automatic)

You don't choose severity - system determines it

Severity Criteria (ANY of these) Impact
CRITICAL
  • Sync Failure (data not reaching Amazon)
  • Data Loss/Corruption
  • Wrong direction at scale (all items affected)
  • Client escalation to management
 Blocks Go decision
Non-Critical Everything else: Logic issues, performance suboptimal, missing data, UI bugs, suggestions Track & improve, doesn't block Go

Issue Resolution Types

Resolved Fixed in code, deployed, verified working
Documented Known limitation - documented, no fix planned
Backlog Will fix later, not blocking rollout

Go/No-Go Decision

Simple two-part criteria

GO if BOTH true:
  1. No Critical issues open
  2. Performance on track (ACOS within target)
NO-GO if EITHER true:
  1. Any Critical issue still open
  2. Performance significantly off target

Known Limitations

Document these as "Documented" not "Resolved"

CVR Estimation Model (Waterfall)

Uses waterfall approach instead of ML. May underestimate or overestimate CVR for keywords with sparse data.

Affected: Bid Optimization accuracy for low-volume keywords

Confidence Levels: Keyword=1.0, Tag=0.8, ASIN=0.6, Default=0.4

3rd Party Keyword Data Source

Jungle Scout data may be incomplete for niche/legacy keywords. Keywords from Harvesting won't have JS data until manually enriched.

Affected: KW Research completeness

Workaround: Manual keyword import for missing terms

Pacing State Persistence

User-locked campaigns (LOCKED_ACTIVE, LOCKED_PAUSED) must be manually unlocked to return to auto-management.

Affected: Campaigns that were manually overridden
