The ASIN Config page is the central hub for managing your products and their advertising configuration.

Target ACOS Hierarchy

The Target ACOS follows a hierarchical override system:

  • ASIN Level (broadest) - applies to all campaigns for this product
  • Campaign Level - overrides ASIN level for specific campaigns
  • Keyword Level (most granular) - overrides campaign level for specific keywords

More specific/granular settings always take precedence over broader settings.

3. Adding a New Product

Step 1: Select Product

Click the Add button on the ASIN Config page to start adding a new product. Select a product from your catalog.

Once selected, the product details will be displayed. Click Next to proceed to competitor selection.

4. Competitor Discovery

After selecting your product, you need to identify competitors. There are two methods:

Method 1: Manual Entry

Manually enter competitor ASINs that you want to target.

Method 2: AI-Powered Discovery (Recommended)

Click Discover Competitors to use AI-powered competitor identification. The system will analyze your product and find relevant competitors.

The AI discovery returns competitors with relevancy scores. You can:

  • Select from the discovered competitors
  • Add manual entries to supplement
  • Mix both methods
Minimum Requirement: At least 5 competitors must be selected to proceed.

5. Keyword Research Workflow

After adding a product with competitors, follow this workflow:

  1. Run KW Research - Click "Generate Research" to start AI-powered keyword research
  2. Check Status - Monitor the "KW Research Status" column for completion
  3. View Research - Once complete, click "View Research" to review and approve the results

The research generates:

  • Keyword suggestions based on your product and competitors
  • Branding scope classifications
  • Keyword-product relationship mappings
  • Attribute rankings
  • Keyword groupings for campaign structure

8. KWs Automation - Tab 3: KWs Grouping & Ranking

This tab organizes keywords into logical groups for campaign creation.

Structure

Three separate tables organized by Branding Scope:

  • Non-Branded (NB) table
  • Own Branded (OB) table
  • Competitor Branded (CB) table

Columns

ColumnDescription
#Row number
Product TypeThe product category (e.g., "Diffuser Aroma Oil")
Variant/Use Case/AudienceSpecific attribute (e.g., "Variant - Italy", "Use Case - Home Scent")
Group # / RankThe group number this attribute combination belongs to
ActionsEdit button for modifications

How Grouping Works

Each group represents a customer search intent cluster:

Example 1: Product Type "Diffuser Aroma Oil" + Variant "Italy" = Group 1

  • All keywords containing "diffuser", "aroma", "oil" AND "Italy" (or synonyms/implied meanings) go into Group 1
  • A campaign will be created for Group 1 containing all these keywords

Example 2: Product Type "Diffuser Aroma Oil" + Variant "Sicilian Lemon, White Rose..." = Group 2

  • Keywords related to these flavor variants go into Group 2
  • A separate campaign will be created for Group 2

Group Numbers by Relationship Type

RelationshipGrouping
Relevant (R)Multiple groups (1, 2, 3, 4...) - further bifurcated by intent
Negative (N)Group 1 only
Substitute (S)Group 1 only
Complementary (C)Group 1 only

Summary Statistics

Shows total keyword counts per branding scope and group number. Helps understand the distribution of keywords across different intent clusters.

Keyword Details Table

Lists individual keywords with:

  • Keyword text
  • Branding Scope
  • Relationship type
  • Group number assignment

After Submit

Once you submit this tab:

  1. System reads the groupings
  2. Creates campaigns based on group numbers
  3. Assigns keywords to their respective campaigns
  4. Campaigns appear in Campaign Config tab

Workflow

  1. Review campaigns - Check the auto-generated campaigns
  2. Set Target ACOS (optional) - Override ASIN-level target for specific campaigns
  3. Set Daily Budget (optional) - Override if not using Auto Budget
  4. Change Status to ENABLED - Select which campaigns to activate
  5. Add Custom Negative KWs (optional) - Exclude specific keywords
  6. Click Submit

Important Notes:

  • First Submit with Status=ENABLED: Campaigns are actually created on Amazon
  • After creation, toggling ENABLED/PAUSED just changes the status
  • The Targeting Spec column helps you decide which campaigns to activate based on the customer intent they target

When It Populates: This tab only shows data AFTER campaigns have been created and enabled for the first time. Before that, it will be empty.

  • Control individual keyword settings

Keyword-Level Overrides

  • Target ACOS: Set a keyword-specific target that overrides campaign and ASIN levels

Workflow

  1. Filter by ASIN or Campaign to focus on specific keywords
  2. Review performance using the metrics columns
  3. Export for bulk modifications
  4. Import modified data back
  5. Submit changes

11. SearchTerm Config

The SearchTerm Config tab provides performance data at the search term level. This is primarily for:

  • Performance review - See which actual search terms are driving results
  • Making modifications - Adjust targeting based on search term performance

This tab is read-focused for analyzing search term data rather than configuration.

Summary: End-to-End Workflow

  1. Add Product (ASIN Config > Add)
  2. Select Competitors (AI Discovery or Manual)
  3. Generate Research (ASIN Config > Generate Research)
  4. Review & Approve Research (KWs Automation Tabs 1-3)
    • Tab 1: Branding Scope & Relationships
    • Tab 2: Listing Attributes
    • Tab 3: Keyword Groupings
  5. Configure Campaigns (Campaign Config)
    • Set Target ACOS, Budget, Status
    • Submit to create campaigns on Amazon
  6. Monitor & Optimize (KWs Config, SearchTerm Config)
    • Review performance
    • Adjust bids and targeting
    • Pause/enable keywords as needed
